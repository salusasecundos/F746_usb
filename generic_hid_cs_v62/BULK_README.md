# STM32 Bulk / WinUSB

Приложение переведено с HID reports на WinUSB Bulk.

## Параметры устройства

- VID: `0x0483`
- PID: `0x5752`
- Device interface GUID: `{8B4B6B6A-3266-4A18-AC3B-7110B602D0A3}`
- Bulk OUT: `0x01`, 64 байта
- Bulk IN: `0x81`, 64 байта

PID `0x5752` выбран специально: Windows запрашивает Microsoft OS-дескриптор
только при первом обнаружении конкретной версии устройства. Новый PID исключает
старый кэш, который мог остаться после прошивки без WinUSB-дескрипторов.

## Запуск

1. Собрать и прошить актуальный STM32-проект.
2. Отключить и снова подключить USB-кабель.
3. Дождаться, пока Windows установит встроенный драйвер `WinUSB`.
4. Запустить `bin\Debug\GenericHid.exe` или собрать решение в Visual Studio.

Сторонняя `libusb.dll` и установка драйвера через Zadig не требуются. В прошивке
добавлены Microsoft OS 1.0 descriptors: они сообщают Windows совместимый ID
`WINUSB` и регистрируют GUID интерфейса для приложения.

Важно: отсутствие ошибки в диспетчере устройств ещё не гарантирует, что выбран
именно `WinUSB`. В свойствах устройства файл драйвера должен быть `Winusb.sys`.
Драйверы `libusbK`, `libusb-win32` и старый HID-драйвер несовместимы с WinUSB API
этого приложения.

Приложение сначала проверяет GUID из прошивки, а затем все реально
зарегистрированные Windows интерфейсы с выбранным VID/PID. При неудаче строка
USB status показывает конкретный этап: интерфейс не зарегистрирован,
`CreateFile`, `WinUsb_Initialize` либо отсутствие Bulk endpoint `0x01/0x81`.

## Формат обмена

Каждый цикл является одной парой запрос/ответ:

1. Приложение отправляет ровно 64 байта в OUT `0x01`.
2. STM32 отвечает ровно 64 байтами через IN `0x81`.

В OUT-пакете нет HID report ID:

- байт `0`: значение `numericUpDown1`;
- байты `1..3`: значения трёх ползунков;
- остальные байты зарезервированы.

В IN-пакете данные BME280 находятся с позиции `5`:

- `5..8`: температура, `int32`, little-endian;
- `9..12`: давление, `uint32`, little-endian;
- `13..16`: влажность, `uint32`, little-endian.

Если Windows ранее уже видела PID `0x5752` с некорректной прошивкой, удалите
`STM32 Bulk Device` в диспетчере устройств вместе с драйвером, затем переподключите
устройство. В нормальном сценарии это не требуется.
